# 2026-07-07 上游差异与 CE 清理结论

## 结论摘要

- 当前复核基准：`upstream/main` 最新提交为 `0f1b03db style: improve switch contrast`。
- CE 版定位：保留 Codex 账号池、本地网关、平台 Key、模型目录、聚合 API、插件中心和本地/服务版运行能力；剥离作者推广、赞助导流、远程 author content、AtomGit 推广等上游个人发行内容。
- 合并策略：只做语义移植，不直接 merge upstream。每个上游提交先拆分为“业务能力 / UI 样式 / 推广内容 / 发行渠道内容”，只吸收符合 CE 定位且能通过当前架构验证的小项。
- 当前状态：低风险功能与性能项已完成语义移植；`09223f6f` / `f3efb3a2` 属大范围产品设计/UI 密度改造，只能拆小项继续评估，不能整包移植。

## CE 版定位

CE 版不是上游个人发行包的镜像，而是面向本地管理和网关转发的社区清理版：

- 保留本地桌面端 + service/web 运行形态。
- 保留账号池、用量观测、平台 Key、模型目录、聚合 API、插件中心和 OpenAI 兼容网关。
- 保留与 Codex/OpenAI 请求链路相关的兼容行为和必要性能优化。
- 删除或不引入与核心能力无关的作者页、赞助推荐、远程作者内容、外部推广入口和发行渠道宣传。
- 对上游 UI 改动按 CE 当前信息密度、性能改造和响应式修复重新评估，不默认继承上游视觉取向。

## 删除 / 不移植项

以下内容与 CE 版定位冲突，已删除或明确不移植：

- 作者页 `/author/`、作者联系方式展示、作者赞助/推荐入口。
- 远程 author content runtime 能力和默认远程内容地址。
- `author.qxnm.top` 相关 Tauri CSP 白名单。
- 专用 sponsor 归一化工具、作者页 Playwright 测试、残留赞助静态资源。
- README 或页面中的 AtomGit 推广、赞助推广、远程 author content 说明。
- 上游含推广内容的整包 README / docs / UI 文案更新。
- `09223f6f` / `f3efb3a2` 中夹带的产品设计重构和推广/作者内容不整包移植；如有价值，只能按页面、组件、交互逐项拆分评估。

## 仍需评估项

### `09223f6f` / `f3efb3a2`

这两组提交属于产品设计重构、桌面布局密度和 UI 细节批量调整，风险不在单个样式，而在整包合并会同时带入：

- CE 已经修过的响应式布局和工具栏换行策略可能被覆盖。
- 作者、赞助、AtomGit、远程 author content 等推广内容可能重新进入。
- 大范围前端 diff 会干扰当前主代理正在处理的 settings 主题预览工作。
- 部分 UI 密度优化需要结合 CE 当前 Dashboard、模型页、设置页和插件中心信息架构重新验收，不能仅以上游视觉结果为准。

建议拆分方式：

1. 先列出每个页面/组件的可观察变化。
2. 剔除推广、作者、发行渠道相关内容。
3. 对 CE 已有响应式修复逐项比对，避免回退中等宽度布局。
4. 每次只移植一个页面或组件的小改动，并跑对应 `tsc` / `node --test` / 必要的浏览器截图验证。

## 为什么不能直接 merge upstream

直接 merge upstream 的主要风险是语义污染，而不是普通冲突成本：

| 维度 | 直接 merge upstream 的问题 | CE 当前策略 |
| --- | --- | --- |
| 产品定位 | 上游包含作者页、赞助导流、AtomGit 推广和远程 author content，和 CE 清理版定位冲突。 | 只保留本地管理、网关、模型与用量相关能力。 |
| 安全与隐私 | 远程 author content / 外部推广域会扩大运行时外联面和 CSP 白名单。 | 删除远程作者内容能力，CSP 不再允许 `author.qxnm.top`。 |
| 可维护性 | 大范围 UI/README/docs 合并会把推广内容、样式重构和真实功能混在一起，审计成本高。 | 按提交语义拆分，小步 cherry-pick / 手工移植。 |
| 回归风险 | CE 已完成大量分页、轻量化、响应式和启动性能改造，整包合并可能覆盖本地优化。 | 对每个功能点按 CE 当前架构重新实现并定向验证。 |
| PR 治理 | 当前 fork 与 upstream 分叉较大，整包 diff 不适合审计，也不利于主代理复核。 | 从干净分支 cherry-pick 关键提交或以 CE 分支小主题提交。 |
