# Windows 登录回调端口回退

## 状态

✅ 已完成

## 现象

`account/login/start` 在 Windows 返回：`以一种访问权限不允许的方式做了一个访问套接字的尝试。 (os error 10013)`。

## 证据

- 登录回调默认监听 `localhost:1455`，与网关 `localhost:48760` 无关。
- 本机没有进程监听 1455。
- Windows IPv4 / IPv6 TCP 排除区间均包含 `1438-1537`，因此 1455 绑定被系统拒绝。
- `v0.3.11` 与当前版本的固定 1455 绑定逻辑一致，不是近期 CE 代码改动引入；系统端口排除区间发生变化后才暴露问题。

## 处理方案

- 默认 1455 绑定失败时回退到 `localhost:0`，由系统选择可用 loopback 端口。
- 用户显式配置 `CODEXMANAGER_LOGIN_ADDR` 时不回退，避免静默偏离配置。
- 回退失败时同时返回固定端口和动态端口两次失败信息。
