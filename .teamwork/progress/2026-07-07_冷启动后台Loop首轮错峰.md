# 冷启动后台 Loop 首轮错峰

## 状态

✅ 已完成

## 背景

`task.md` 中剩余 P2 性能观察项要求处理冷启动首轮后台 loop 错峰。当前服务启动会连续启动 usage polling、gateway keepalive、token refresh polling、warmup cron 等后台线程，其中前三类动态轮询会在进入 loop 后先执行任务再 sleep，容易在冷启动瞬间叠加 DB 与网络压力。

## 处理内容

- 在 usage refresh 后台 loop 启动包装层加入首轮固定错峰延迟。
- 错峰顺序：
  - usage polling：5 秒
  - gateway keepalive：15 秒
  - token refresh polling：25 秒
  - warmup cron：35 秒
- 仅延迟线程启动后的首轮执行，不改变后续轮询间隔、失败退避、jitter 或业务调度配置。
- 延迟期间每秒检查 shutdown 标记，避免服务关闭时长期等待。
- 如果服务在错峰等待期间进入 shutdown，后台线程会直接退出，不再进入首轮 worker。
