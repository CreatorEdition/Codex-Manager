# 请求体 JSON 解析收敛

## 状态

✅ 已完成

## 背景

`task.md` 中剩余 P2 性能观察项包含“请求体多次 JSON parse 收敛”。子代理只读扫描指出本地校验阶段会对同一 body 分别执行 service tier 诊断、metadata 提取与文本长度校验，多候选上游尝试也会重复解析原始 body 提取 `prompt_cache_key`。

## 本轮处理内容

- `request_helpers` 新增基于已解析 `serde_json::Value` 的 helper，保留原 body 入参兼容函数。
- `build_local_validation_result` 原始 body 阶段复用同一次 JSON 解析结果生成 service tier 诊断与初始 metadata。
- 最终归一化 body 阶段复用同一次 JSON 解析结果执行文本长度校验与最终 metadata 提取。
- `CandidateExecutionState` 对原始 body 的 `prompt_cache_key` 提取做单请求内缓存，并用缓存的 `Bytes` 内容确认防止不同 body 误复用。

## 审计备注
