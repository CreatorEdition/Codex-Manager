# 2026-07-07 Web 启动设置读取优化

## 执行方

- 【CodeX-GPT】主代理实施与审计。

## 目标

- 语义移植上游 `0508c19a`，解决 Web gateway 冷启动读取设置可能因 15 秒默认超时失败的问题。
- 同时收敛 `appSettings/get` 的设置快照写放大，避免 Web 启动/刷新时无变化也刷新整批 `app_settings.updated_at`。

## 实现

- 前端 `AppBootstrap` 新增 `startupAppSettingsTimeoutMs()`：仅 `web-gateway` 模式使用 60 秒设置读取超时，其他模式继续使用 15 秒。
- 新增 `apps/tests/app-bootstrap-startup.test.mjs`，并纳入 `test:runtime` 脚本。
- 后端拆分 `current_app_settings_value()` 与 `current_app_settings_value_persisted()`：
  - `appSettings/get` 使用按变化写入快照。
  - `appSettings/set` 仍使用完整持久化快照，保留设置写入后的同步语义。
- `appSettings/get` 对已存在且值未变化的设置不再调用 `save_persisted_app_setting()` / `save_env_overrides_value()`，但仍会补种缺失快照、清理 env override 保留键和规范化异常旧值。
