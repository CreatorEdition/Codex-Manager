# 2026-07-07 Dialog 布局定位修复

## 执行方

- 【CodeX-GPT】主代理实施与审计。

## 目标

- 语义移植上游 `6ac01a2a fix: correct dialog layout positioning`。
- 跳过上游 README 中 AtomGit/赞助相关推广内容，只保留 CE 需要的弹窗布局修复。

## 实现

- `DialogContent` 不再使用 fixed + translate 居中；新增 `DialogViewport` 负责全屏 fixed、居中和纵向滚动。
- Dialog 主体改为相对定位并使用 `100dvh` 高度约束，降低小高度窗口下内容被裁切的概率。
- 平台密钥弹窗改为固定头部/底部、内容区独立滚动，避免长表单按钮区被卷走或弹窗整体超出视口。
- `Button` 组件改为 `cn(buttonVariants({ variant, size }), className)`，确保调用方 class 在 `tailwind-merge` 中后置生效。
- 新增 `apps/tests/dialog-layout.test.mjs` 并纳入 `test:runtime`。
