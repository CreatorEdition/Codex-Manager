# 2026-07-07 桌面启动与运行渲染性能项

## 执行方

- 【CodeX-GPT】主代理实施与审计。

## 目标

- 语义移植上游 `03ca2052 perf: optimize desktop startup and runtime rendering` 中适合 CE 当前架构的低风险部分。
- 避免整包合入上游 before-build 代理重构、锁文件变更和后续产品设计重构内容。

## 实现

- `apps/next.config.ts` 仅在开发服务器阶段增加 webpack watch 忽略规则：`node_modules`、`src-tauri/target`、`.pnpm-store`，并设置 1000ms poll。
- `.gitignore` 增加 `/.pnpm-store/`，避免本地 pnpm store 混入工作区。
- `useAppStore` 增加 patch 变化判断和 tabs 等价判断；无变化的设置、服务状态、运行能力、侧边栏状态和引导弹窗状态更新会返回原状态。
- `AppBootstrap`、Header、Sidebar、日志页改为按字段 selector 订阅 store，降低全局 store 局部变更导致的重渲染范围。
- `useDashboardStats` 支持 `includeApiModels` 参数；托盘预览传 `false`，避免预览窗口拉取不需要的 API 模型目录。

## 未移植说明

- 上游 `before-build.mjs` 的 startup page 等待与代理重构不适用于 CE 当前脚本结构；CE 当前脚本已经采用可复用 dev server / stale lock 清理路径。
- 上游 startup snapshot 新增的若干 include 参数与 CE 当前 `accountLimit`、`apiKeyLimit`、`includeUsageAggregate`、`includeTodaySummary` 体系不完全一致；本轮保留 CE 现有轻量快照协议。
